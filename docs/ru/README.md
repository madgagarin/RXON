**RXON** (Reverse Axon) — это легковесный расширяемый протокол обратного соединения, разработанный для архитектур **[HLN (Hierarchical Logic Network)](https://github.com/avtomatika-ai/hln)**.

Он служит «нервной системой» для распределенных мультиагентных систем, предоставляя строго типизированный фундамент Zero Trust для межсервисного взаимодействия.

## 🚀 Концепция

В традиционных сетях команды обычно передаются «сверху вниз» (Push-модель). В **RXON** инициатива соединения всегда исходит от подчиненного узла (Shell) к вышестоящему (Оркестратору). Эта архитектура «обратного аксона» позволяет воркерам работать за NAT или файрволами без сложной настройки сети, сохраняя при этом защищенный двусторонний канал управления.

## ✨ Ключевые особенности

-   **Обратное соединение (PULL)**: Узлы подключаются к оркестратору для получения задач, что обеспечивает совместимость со сложными сетевыми окружениями (NAT/Firewalls).
-   **Безопасность Zero Trust**: Подпись полезной нагрузки через HMAC-SHA256 с верификацией за константное время. Поддержка цепочек идентичности и извлечение данных mTLS-сертификатов.
-   **Глубокое восстановление моделей**: Мощная утилита `from_dict` рекурсивно восстанавливает сложные типы Python (`NamedTuple`, `dataclass`, `Enum`, `UUID`) из сырых словарей, поддерживая вложенность и типы `Union`.
-   **Безопасная сериализация**: Утилита `to_dict` рекурсивно удаляет `None` для уменьшения размера и нормализует `float` (например, `1.0` -> `1`), обеспечивая стабильность криптографических хешей.
-   **Автоматическая валидация контрактов**: Встроенный движок JSON Schema автоматически выводит схемы из типов Python и проверяет параметры `TaskPayload` на соответствие контрактам `SkillInfo`.
-   **Продвинутое сопоставление ресурсов**: Математическая логика распределения задач:
    *   **Числа**: Используется логика **GE (Greater or Equal)** (Требование <= Доступно).
    *   **Списки**: Используется логика **Вхождения** (значение в списке) или **Пересечения** (общие элементы).
    *   **Строки**: Частичное совпадение моделей оборудования без учета регистра.
-   **Универсальная телеметрия**: Хартбиты включают детальные метрики для любых кастомных устройств (датчики, GPU, приводы) и общие системные свойства через расширяемую модель `HardwareDevice`.
-   **Отказоустойчивый транспорт**: Реализация HTTP/WebSocket с автоматическим обновлением токенов (STS), экспоненциальным backoff при переподключениях и корректным завершением сессий.

## 🏗 Архитектура и Логика

### Внутренняя валидация и нормализация
Библиотека обеспечивает целостность данных на нескольких уровнях:
1.  **Стабильность сериализации**: При подписи сообщений RXON нормализует числовые типы и сортирует ключи словарей, чтобы один и тот же объект всегда давал одинаковый HMAC-хеш.
2.  **Защита от рекурсии**: Все рекурсивные операции ограничены глубиной 100 для предотвращения переполнения стека или DoS-атак через вредоносные данные.
3.  **Соблюдение схем**: Перед выполнением задачи библиотека проверяет входные параметры на соответствие JSON-схеме навыка, проверяя обязательные поля, типы и разрешенные значения (Enums).

### Логика умного сопоставления (Smart Matching)
RXON формализует правила подбора исполнителя для задачи:
1.  **Сопоставление железа**: Сравниваются свойства `HardwareDevice`. Если задаче требуется `vram_gb: 16`, она подойдет любому устройству с `vram_gb >= 16`.
2.  **Свойства ресурсов**: Общие ресурсы (такие как RAM или ядра CPU) сопоставляются через словарь `properties` с использованием той же логики GE.
3.  **Пересечение возможностей**: Если задача допускает несколько ОС (например, `["linux", "darwin"]`), воркер с `linux` будет успешно сопоставлен.

## 🛡️ Обработка ошибок

RXON определяет набор стандартизированных кодов ошибок для обеспечения единообразного поведения во всех реализациях:
-   `CONTRACT_VIOLATION_ERROR`: Данные не соответствуют согласованной схеме.
-   `SECURITY_ERROR`: Ошибка аутентификации или проверки подписи.
-   `RESOURCE_EXHAUSTED_ERROR`: Недостаточно физических ресурсов (RAM, VRAM).
-   `DEPENDENCY_ERROR`: Требуемый сервис или артефакт недоступен.

## 🧪 Быстрый старт

```python
from rxon import create_transport
from rxon.models import Resources, HardwareDevice

# 1. Создание транспорта (поддерживает http, https, ws, wss)
transport = create_transport("ws://api.hln.local", "worker-01", "secret-token")

# 2. Определение ресурсов воркера (RAM и ядра CPU теперь в properties)
my_res = Resources(
    properties={"ram_gb": 64, "cpu_cores": 16},
    devices=[HardwareDevice(type="gpu", model="RTX 4090", properties={"vram_gb": 24})]
)

# 3. Логика сопоставления (Требование: GPU минимум 16GB VRAM)
req = Resources(devices=[HardwareDevice(type="gpu", properties={"vram_gb": 16})])
if my_res.matches(req):
    print("Этот холон готов к выполнению задачи!")
```
