**RXON** (Reverse Axon) — это легковесный расширяемый протокол межсервисного взаимодействия с обратным соединением, разработанный для архитектур **[HLN (Hierarchical Logic Network)](https://github.com/avtomatika-ai/hln)**.

Он служит «нервной системой» для распределенных мультиагентных систем, обеспечивая строго типизированную основу Zero Trust для взаимодействия холонов.

## 🚀 Концепция

В традиционных сетях команды обычно передаются «сверху вниз» (Push-модель). В **RXON** инициатива соединения всегда исходит от подчиненного узла (Shell) к вышестоящему (Оркестратору). Эта архитектура «обратного аксона» позволяет воркерам работать за NAT или файрволами без сложной настройки сети, сохраняя при этом защищенный двусторонний канал управления.

## ✨ Основные возможности

-   **Обратное соединение (PULL)**: Узлы подключаются к оркестратору для получения задач, что обеспечивает совместимость со сложными сетевыми окружениями.
-   **Безопасность Zero Trust**: Встроенная поддержка цифровых подписей (`SecurityContext`) и цепочек идентичности. Все сообщения могут быть криптографически проверены через множество уровней холархии.
-   **Агностический и расширяемый дизайн**: Базовые модели (ресурсы, навыки, задачи) полностью расширяемы через универсальные поля `metadata` и `properties`, что делает протокол подходящим для ИИ, IoT и робототехники.
-   **Универсальная телеметрия**: Хартбиты включают детальные метрики CPU, RAM и любых кастомных устройств (датчики, GPU, приводы) через расширяемую модель `HardwareDevice`.
-   **Универсальная система событий**: Единый механизм сигнализации для обновлений прогресса, кастомных алертов и триггеров реального времени с иерархическим всплытием событий (bubbling).
-   **Умное сопоставление ресурсов (Smart Matching)**: Формализованная логика для требований к оборудованию. Поддерживает логику **GE (Greater or Equal)** для чисел и частичное совпадение строк для моделей.
-   **Подключаемые Blob-хранилища**: Стандартизированный интерфейс `BlobProvider` для выноса тяжелых данных через S3, GCS или локальные хранилища.
-   **Единая фабрика**: Простая инициализация транспорта через `create_transport` с поддержкой схем `http://`, `https://`, `ws://` и `wss://`.
-   **Ядро без зависимостей**: Ядро протокола написано на чистом Python 3.11+. Стандартные транспорты используют `aiohttp` и `orjson` для максимальной производительности.

## 🏗 Архитектура

Протокол разделен на два основных интерфейса:

1.  **Transport (сторона воркера)**: Для инициирования соединений, получения задач, отправки событий и результатов.
2.  **Listener (сторона оркестратора)**: Для приема входящих соединений и маршрутизации сообщений в движок.

### Логика умной диспетчеризации

RXON формализует правила сопоставления задач и холонов:
1.  **Совпадение по ID**: Прямое соответствие по идентификатору устройства.
2.  **Тип и модель**: Точное совпадение типа и частичное совпадение строки модели (case-insensitive).
3.  **Сопоставление свойств (Smart Comparison)**:
    *   **Числа**: Проверка «не менее чем» (Значение воркера >= Требование).
    *   **Остальное**: Строгое равенство.

## 🛡️ Обработка ошибок

RXON определяет набор стандартизированных кодов ошибок для обеспечения единообразного поведения во всех реализациях:
-   `CONTRACT_VIOLATION_ERROR`: Данные не соответствуют согласованной схеме.
-   `SECURITY_ERROR`: Ошибка аутентификации или проверки подписи.
-   `RESOURCE_EXHAUSTED_ERROR`: Недостаточно физических ресурсов (RAM, VRAM).
-   `DEPENDENCY_ERROR`: Требуемый сервис или артефакт недоступен.

## 🧪 Быстрый старт

```python
from rxon import create_transport
from rxon.models import Resources, HardwareDevice

# 1. Создание транспорта (поддерживает http, https, ws, wss)
transport = create_transport("ws://api.hln.local", "worker-01", "secret-token")

# 2. Определение ресурсов воркера
my_res = Resources(
    cpu_cores=8,
    devices=[HardwareDevice(type="gpu", model="RTX 4090", properties={"memory_gb": 24})]
)

# 3. Логика умного сопоставления
req = Resources(cpu_cores=4, devices=[HardwareDevice(type="gpu", properties={"memory_gb": 16})])
if my_res.matches(req):
    print("Этот холон готов к выполнению задачи!")
```
